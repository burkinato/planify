# General Rules

## Communication Efficiency
- **Be Concise**: Skip pleasantries; focus on technical solutions.
- **High Density**: Use bullet points and markdown tables for complex data.
- **No Redundancy**: Don't repeat code blocks unless changes are made.
- **Selective Context**: Only read files that are strictly relevant to the task.

## Code Quality
- **Type Safety**: Prefer TypeScript; avoid `any`.
- **Consistency**: Follow existing patterns in `useEditorStore` and `EditorCanvas`.
- **Clean Diffs**: Provide precise instructions for code modifications.
